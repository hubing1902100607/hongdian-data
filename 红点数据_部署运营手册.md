## 一、首次部署（只做一次，约15分钟）

### 第1步：注册 GitHub 账号
1. 打开 https://github.com，点右上角 **Sign up**
2. 填邮箱、密码、用户名（比如 `hongdian-data`），完成验证
3. 登录后进入主页

### 第2步：新建仓库
1. 点右上角 **+** → **New repository**
2. Repository name 填：`hongdian-data`
3. 选 **Public**（必须选 Public，GitHub Pages 免费版只支持公开仓库）
4. 勾选 **Add a README file**
5. 点 **Create repository**

### 第3步：上传文件
1. 进入刚创建的仓库页面
2. 点 **Add file** → **Upload files**
3. 把以下文件全部拖进去：
   - `xhs_tracker.html`
   - `membership.html`
   - `红点数据_导入模版.csv`
   - `alipay.png`（你的支付宝收款码，上传前重命名为这个）
4. 在下方 Commit changes 框里随便填个说明，比如"初始上线"
5. 点 **Commit changes**

### 第4步：开启 GitHub Pages
1. 点仓库页面上方的 **Settings** 标签
2. 左侧菜单找到 **Pages**
3. Source 选 **Deploy from a branch**
4. Branch 选 **main**，文件夹选 **/ (root)**
5. 点 **Save**
6. 等待约 1 分钟，页面会显示你的网址：
   ```
   https://hubing1902100607.github.io/hongdian-data/
   ```

### 第5步：确认网址可访问
- 数据页：`https://hubing1902100607.github.io/hongdian-data/xhs_tracker.html`
- 会员页：`https://hubing1902100607.github.io/hongdian-data/membership.html`
- 把数据页网址发给自己测试登录

> **重要**：把这两个网址记下来，会员页网址可以直接发给潜在用户。

## 二、日常运营操作

### 2.1 给新用户发兑换码

**第1步：生成兑换码**

格式建议：`RD-年份-随机4位字母数字`，比如 `RD-2025-K3M7`

可以自己随便想，也可以用这个规律：
```
月度码：RD-M-XXXX  （M = Monthly）
季度码：RD-Q-XXXX  （Q = Quarterly）
年度码：RD-Y-XXXX  （Y = Yearly）
```

**第2步：在代码里添加这条码**

1. 打开 GitHub 仓库，点 `xhs_tracker.html`
2. 右上角点铅笔图标（Edit this file）
3. 按 **Ctrl+F**，搜索 `REDEEM_CODES`，找到这段代码
4. 在 `// ── 付费码 ──` 下方加一行：

```js
// 月度（30天）
'RD-M-K3M7': { expiry: '2025-04-19', issuedAt: '2025-03-19T14:30', label: '张三 · 月度' },

// 季度（90天）
'RD-Q-A2B9': { expiry: '2025-06-19', issuedAt: '2025-03-19T14:30', label: '李四 · 季度' },

// 年度（365天）
'RD-Y-C5D1': { expiry: '2026-03-19', issuedAt: '2025-03-19T14:30', label: '王五 · 年度' },
```

> **issuedAt 说明**：填你当前的时间，格式 `YYYY-MM-DDTHH:mm`。
> 用户必须在这个时间后的 **24小时内** 首次激活，否则码自动失效。

**到期日速算：**
| 套餐 | 到期日计算 |
|---|---|
| 月度 ¥59.9 | 今天日期 + 30天 |
| 季度 ¥135 | 今天日期 + 90天 |
| 年度 ¥365 | 今天日期 + 365天 |

5. 点 **Commit changes** → 约10秒后生效

**第3步：把码发给用户**

直接私信发送兑换码，附上使用说明：
```
您好！以下是您的红点数据兑换码：

兑换码：RD-Q-A2B9
有效期至：2025-06-19
访问地址：https://hubing1902100607.github.io/hongdian-data/xhs_tracker.html

⚠️ 请在24小时内激活，激活后绑定当前设备。
```

### 2.2 每周更新数据

1. 按照 `红点数据_导入模版.csv` 的格式填写本周数据
2. 以管理员密码登录数据页
3. 点右上角 **📥 导入销量数据**
4. 上传 CSV 文件，预览确认后点 **确认导入**

> 更新完成后，在公众号「北苑笔谈」发一条通知，告知用户本周数据已更新。

---

### 2.3 封禁或停用某个用户

1. 打开 GitHub，编辑 `xhs_tracker.html`
2. 找到对应的兑换码那一行，直接删掉或改 expiry 为过去的日期：
```js
// 改成过去的日期即可立即失效
'RD-Q-A2B9': { expiry: '2020-01-01', issuedAt: '...', label: '李四 · 已停用' },
```
3. Commit changes，约10秒生效

---

### 2.4 用户反馈换设备（设备损坏/换电脑）

目前版本无法服务器端重置，处理方式：
1. 让用户截图付款凭证确认是本人
2. 在代码里把旧码 expiry 改为过期
3. 新建一条新码（相同到期日），重新发给用户

## 五、后续升级路线（备忘）

当付费用户超过 **50人** 时，建议升级到 Cloudflare Workers 后端：

- 兑换码存服务器端 → 用户无法通过看源码获取所有码
- 设备绑定存服务器端 → 清缓存、换浏览器都无法绕过
- 可以看到实时用户登录记录
- 操作仍然免费（Cloudflare Workers 免费额度足够）
